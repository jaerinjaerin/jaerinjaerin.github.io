## `Partial<T>`

<Callout type="info">
**Partial의 정의**

Partial는 제네릭 타입으로, 타입 T의 모든 속성을 선택적(optional)으로 만든다.

</Callout>

---

## Partial의 타입 사용 예시

### 1. 속성 optional로 만들기

User의 타입을 지정했다고 가정해보자.

```typescript
interface User {
  id: string;
  email: number;
  phoneNum: number;
}
```

이때 phoneNum이 없는 사용자가 있다면 Partial을 사용해 타입의 속성을 선택적으로 사용할 수 있다.

```typescript
const userWithPhone: Partial<User> = {
  id: 1,
  email: 'someEmail@email.com',
};
```

<Callout type="success">
**Partial의 장점**

모든 속성이 선택적이 되므로, 필요한 속성만 제공할 수 있다. 이는 부분적인 데이터 업데이트나 선택적 속성이 필요한 경우에 유용하다.

</Callout>

### 2. 함수의 매개변수로 사용

```typescript
interface Product {
  id: number;
  name: string;
  price: number;
  description?: string;
}

const updateProduct = (
  product: Product,
  updates: Partial<Product>
): Product => {
  return { ...product, ...updates };
};

let product: Product = { id: 1, name: 'Laptop', price: 1000 };

// price만 업데이트
product = updateProduct(product, { price: 900 });

console.log(product);
```

<Callout type="info">
**함수 매개변수에서의 활용**

`updates` 매개변수를 `Partial<Product>`로 지정하면, Product의 모든 속성 중 일부만 전달할 수 있다. 이는 객체의 부분 업데이트를 구현할 때 매우 유용하다.

</Callout>

### 3. 객체의 부분 업데이트

```typescript
interface Profile {
  username: string;
  bio: string;
  website?: string;
}

// 기존 Profile 타입을 Partial을 사용하여 모든 속성을 optional로 만든다
type PartialProfile = Partial<Profile>;

const updateProfile = (
  profile: Profile,
  updates: Partial<Profile>
): Profile => {
  return { ...profile, ...updates };
};

let profile: Profile = { username: 'john_doe', bio: 'Web Developer' };

// bio와 website만 업데이트
profile = updateProfile(profile, {
  bio: 'Full Stack Developer',
  website: 'https://example.com',
});

console.log(profile);
```

<Callout type="success">
**실전 활용 시나리오**

Partial은 다음과 같은 상황에서 특히 유용하다:

- 폼 데이터의 부분 업데이트
- API 요청에서 선택적 파라미터 처리
- 상태 관리에서 부분 상태 업데이트
- 설정 객체에서 일부 옵션만 오버라이드

</Callout>

## 정리

<Callout type="info">
**Partial 타입의 핵심**

`Partial<T>`는 TypeScript의 유틸리티 타입 중 하나로, 타입의 모든 속성을 선택적으로 만들어준다. 이를 통해 타입 안정성을 유지하면서도 유연한 객체 업데이트가 가능하다.

</Callout>
